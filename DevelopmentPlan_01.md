# DevelopmentPlan_01.md

## 需求描述：模式切换下光标精准定位

### 背景
用户在编辑模式和Markdown预览模式之间切换时，期望无论内容如何变化，光标始终准确保持在原来写作内容的相对语义位置，不出现跳行、错位、偏移等问题。

#### 典型场景
- 光标当前位置：“### 三号标题”的“题”字后面。转换为预览再返回，仍然位于同样位置。
- 涉及内容含有标题、列表、粗体、表格、emoji等情况下，也需保持一致体验。

---

## 解决方案

### 1. 记录并传递光标“锚点”信息
- 模式切换离开编辑区时，除记录字符 offset，还应提取当前行内容、前后若干字符和 Markdown 块级结构（如段落、标题等上下文）。
- 可选：存储选区起止、当前行列等。

### 2. 渲染插入锚点，关联光标定位
- Markdown 转 HTML 时，为文本块（如标题、段落）动态插入 `data-anchor` 属性（如内容哈希+字符 offset）。
- 渲染完成后，自动滚动并聚焦到锚点处，确保光标附近文本高亮显示。

### 3. 反向还原
- 从预览回到编辑区时，同理以锚点还原光标 offset、行/列，恢复到精准编辑位置。

### 4. 兼容复杂结构和异常场景
- 若某些复杂语法（如表格、嵌套列表等）导致不能精确映射，优先回退至本行或主段落开头。
- 多选区、多光标高亮时同步遍历映射。

### 5. 性能与模块化
- 锚点定位算法可异步处理，不影响UI流畅（如 setTimeout 20ms 后执行）。
- 封装专用 anchor/util 模块，便于单元测试及维护。

### 6. 推荐子任务拆解
1. 提取编辑区锚点信息的工具方法（如 getAnchorFromEditor）
2. Markdown 渲染插入 data-anchor 属性并滚动定位
3. 模式切换统一调用锚点保存/还原
4. 完善典型和边界用例自动化测试
5. 性能优化与异常回退机制

当前光标字符 offset
当前行内容（或前后 n 字文本片段）
行号、在 Markdown 结构所在的块（如段落、列表、标题）信息

☐ 渲染完成后利用 JS 定位带锚点的元素，聚焦/滚动至该 anchor 区域。

☐ 在切回编辑区时，根据 anchor 信息还原编辑区光标为原 offset 或最近可用的 fallback 行/列。
☐ 保证复杂 Markdown 渲染（如多行标题、表格、列表等）时 anchor 定位的健壮性，失败时回退至同行/首行。
4. 工具与重构
☐ 新增/重构 anchor util 工具模块，实现文本/DOM锚点的映射与查找。
☐ 优化模式切换主流程，统一调用锚点保存、恢复逻辑。
5. 自动化测试及用例
☐ 针对典型和复杂 Markdown 文本，编写自动化测试用例，验证 anchor 能否稳定还原光标。
☐ 常见异常场景（如内容边界、多重块级结构、特殊符号等）补充覆盖。
6. 性能与回退
☐ anchor 定位逻辑允许异步/延时处理（如 setTimeout），确保 UI 流畅
☐ 显示锚点不能精确还原时，落至本段起始或文本结尾。
