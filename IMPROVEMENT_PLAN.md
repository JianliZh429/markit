# IMPROVEMENT_PLAN.md

## 1. 安全性与健壮性

- [ ] **完善路径安全机制**  
  - 解决空目录路径合法性判断 bug（参考 DEVELOPMENT_PLAN.md 1.1）。
  - 针对符号链接与目录穿越加强校验，防止绕过文件系统限制。
  - 强制保存/打开文件仅支持 `.md` 等 Markdown 合法后缀。

- [ ] **IPC 与主/渲染进程通信安全**  
  - 避免开放多余 Node 能力到渲染进程，确保 contextBridge 白名单准确。
  - 对所有主进程公开的 API 做最小权限控制。

- [ ] **粘贴内容 HTML → Markdown 时进行 XSS 过滤**  
  - 增加如 DOMPurify 的 HTML 消毒过程，避免恶意脚本注入。

## 2. 性能与大文件/大目录支持

- [ ] **文件树/目录懒加载**  
  - 大型目录仅动态加载展开目录，提升 UI 响应速度。

- [ ] **高并发下的文件搜索优化**  
  - `search-in-files` 用 Promise.all 并发并设置并发上限，避免阻塞事件循环。
  - 提供 abort/cancel 支持，提升全局搜索的可控性。

- [ ] **LRU 缓存健壮性与内存使用优化**  
  - 延展 MarkDown 渲染结果、搜索结果的缓存策略。

## 3. 类型和状态管理

- [ ] **严格 TypeScript 配置**  
  - `tsconfig.json` 启用`strict`模式，全面校正类型声明。
  - 为 `stateManager`、核心服务接口添加/优化接口定义。

- [ ] **模块 import/export 一致性**  
  - ESM/CJS 混用时保持一致，防止运行时导入导出错误。

## 4. 用户体验改进

- [ ] **优化编辑模式和Markdown预览模式，光标在文字内容的位置跳到**
  - 目前，在切换模式后，光标会移动到文字的不同位置，比如：文中内容为， “### 三号标题” 光标在“题”字后面， 切换预览后，光标可能就会跳动到其他行了

- [ ] **优化未保存内容关闭提示**  
  - 实现检测未保存修改，关闭窗口前提示用户保存。

- [ ] **主题、字体、UI 个性化配置**  
  - 支持 JSON 格式配置/切换明暗主题、字体大小。

- [ ] **导出功能扩展（PDF/HTML）**  
  - 利用 Electron `printToPDF` 或 HTML 文件导出。

- [ ] **Undo/Redo 栈持久化**  
  - 支持跨会话撤销/重做历史本地存储。

## 5. 测试与 DevOps

- [ ] **安全模块与关键功能单元测试补齐**  
  - 特别是目录验证、IPC 通信、文件操作等边界测试。

- [ ] **CI/CD 流程健全**  
  - 集成自动测试、代码规范检测（ESLint），并实现 pre-commit 钩子。

## 6. 文档和团队协作
